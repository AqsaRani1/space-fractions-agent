# Space Fractions

## Description
Space Fractions is an educational web game designed for 6th grade students to practice their fraction skills. The game features a space-themed environment and challenges players with a variety of fraction-based questions and exercises.

## Architecture Overview
The application is built using a microservices architecture, with the following three services:

| Service        | Description |
| -------------- | ----------- |
| GameComponent | Handles the game logic, user interactions, and score tracking. |
| QuestionComponent | Generates and provides the fraction-based questions and exercises. |
| UserComponent | Manages user accounts, authentication, and leaderboard functionality. |

## Technology Stack
- **Frontend**: React, TypeScript, Webpack, Sass
- **Backend**: Node.js, Express, TypeScript
- **Database**: MongoDB
- **Deployment**: Docker, Kubernetes
- **Testing**: Jest, Enzyme, Supertest

## Prerequisites
- Node.js (version 14 or higher)
- MongoDB (version 4.4 or higher)
- Docker (version 20.10 or higher)

## Setup
1. Clone the repository: `git clone https://github.com/your-username/space-fractions.git`
2. Install dependencies: `cd space-fractions && npm install`
3. Set up the environment variables:
   - Create a `.env` file in the root directory
   - Add the required environment variables, such as database connection strings and API keys
4. Build and run the Docker containers:
   - `docker-compose up --build`

## Running the Application
1. Once the setup is complete, the application will be available at `http://localhost:3000`.
2. Users can sign up, log in, and start playing the game.
3. The leaderboard feature allows users to view the top-performing players.

## API Endpoints
| Endpoint         | Method | Description |
| ---------------- | ------ | ----------- |
| `/api/questions` | GET    | Retrieve a list of fraction-based questions. |
| `/api/users`     | POST   | Create a new user account. |
| `/api/users/login` | POST | Authenticate a user and obtain a session token. |
| `/api/leaderboard` | GET | Retrieve the top-scoring users. |

## Running Tests
1. Run the unit tests: `npm run test`
2. Run the integration tests: `npm run test:integration`
